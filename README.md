# inVision U -- Система интеллектуального отбора кандидатов

Платформа для приёмной комиссии инновационного университета **inVision U** (проект inDrive, Казахстан). Система реализует объяснимый ИИ-скоринг для многомерной оценки и ранжирования кандидатов с NLP-анализом эссе, детекцией ИИ-генерации, анализом справедливости и валидацией модели.

## 1. Описание проекта

inVision U -- интеллектуальная веб-платформа для автоматизации процесса отбора кандидатов в образовательную программу. Система обеспечивает прозрачную, объяснимую и справедливую оценку соискателей по пяти ключевым измерениям, используя NLP-анализ текстов, эвристическую детекцию ИИ-генерации и механизм корректировки оценок комиссией (human-in-the-loop).

---

## 2. Проблема

Приёмные комиссии сталкиваются с большим потоком заявок, субъективностью оценки, отсутствием прозрачности в принятии решений и невозможностью масштабировать экспертную оценку. Ручной отбор приводит к потере сильных кандидатов, непоследовательности критериев и невозможности аудита решений. Дополнительно, рост использования генеративного ИИ для написания мотивационных писем усложняет оценку подлинности заявок.

---

## 3. Решение

Система inVision U предоставляет:

- Многомерную оценку кандидатов по 5 измерениям с настраиваемыми весами
- NLP-анализ мотивационных эссе: тональность, сложность, аутентичность, лексическое богатство
- Эвристическую детекцию ИИ-генерации текстов
- Объяснимость каждого решения с детализацией суб-компонентов и доказательной базой
- Автоматическое формирование шорт-листа с порогами
- Механизм корректировки оценок комиссией (human-in-the-loop)
- Анализ справедливости по демографическим группам (город, образование, опыт)
- Валидацию модели: стратифицированное разделение, абляционное исследование, анализ чувствительности
- Сравнение с тремя базовыми моделями отбора
- Экспорт результатов в JSON и CSV
- Режим анонимизации персональных данных

## 6. Методология скоринга

### 5 измерений оценки

| Измерение | Вес по умолчанию | Суб-компоненты | Источники данных |
|-----------|:---:|------|------|
| Мотивация | 25% | Анализ эссе (45%), волонтёрство (20%), рекомендации (15%), проекты (20%) | Эссе (NLP-ключевые слова, тональность), волонтёрские часы, кол-во рекомендаций, проекты |
| Лидерство | 20% | Роли (35%), эссе (30%), командные достижения (20%), волонтёрское лидерство (15%) | Лидерские роли, NLP-анализ эссе, категории достижений |
| Траектория роста | 20% | GPA (25%), эссе (30%), уровень образования (20%), достижения (25%) | GPA, education_level, NLP-анализ эссе, разнообразие достижений |
| Навыки | 20% | Широта навыков (30%), ключевые навыки (35%), языки (15%), проекты (20%) | Список навыков, сверка с HIGH_VALUE_SKILLS, языки, проекты |
| Опыт | 15% | Стаж работы (35%), достижения (25%), волонтёрство (20%), проекты (20%) | Годы опыта, достижения, волонтёрские часы, проекты |

### NLP-модификаторы

- **NLP-бонус**: начисляется за высокое качество эссе (сложность текста, аутентичность, лексическое богатство). Коэффициент настраивается (по умолчанию 10%).
- **Штраф за ИИ-генерацию**: применяется при обнаружении высокой вероятности автоматической генерации текста (по умолчанию 15%).

### Детекция ИИ-генерации

Эвристический анализ включает:
- Однородность длины предложений (низкая вариативность указывает на ИИ)
- Плотность переходных/связующих слов (ИИ-тексты используют их чрезмерно)
- Уникальность биграмм (ИИ-тексты имеют более предсказуемые паттерны)
- Наличие маркеров человеческого авторства (личные местоимения, эмоциональные выражения, конкретные детали)

## 7. Базовые модели и сравнение

Система сравнивает результаты ИИ-скоринга с тремя наивными подходами для демонстрации добавленной ценности:

| Метод | Описание | Типичный результат |
|-------|----------|----|
| **Случайный отбор** | Выбор N кандидатов случайным образом | Низкий средний балл, потеря сильных кандидатов |
| **Только по GPA** | Сортировка по GPA, выбор топ-N | Игнорирует мотивацию, лидерство, навыки; теряет кандидатов с высоким потенциалом, но средним GPA |
| **Равные веса** | Все 5 измерений с весом 0.2 | Не учитывает приоритеты организации; может завышать или занижать значимые критерии |

Для каждого метода рассчитываются: средний балл топ-N, минимальный балл, количество потерянных сильных кандидатов (присутствующих в ИИ-топ-N, но отсутствующих при данном методе).

---

## 8. Валидация модели

### Стратифицированное разделение (70/30)

Кандидаты разделяются на обучающую (70%) и тестовую (30%) выборки. Для обеих рассчитываются: среднее, медиана, стандартное отклонение, минимум, максимум. Малая разница между выборками подтверждает стабильность модели.

### Абляционное исследование

Для каждого из пяти измерений выполняется исключение (zeroing out) с перераспределением весов на оставшиеся. Анализируется изменение среднего балла, что показывает относительный вклад каждого измерения.

### Анализ чувствительности весов

Веса каждого измерения сдвигаются на +/-10% с компенсацией на остальных. Оценивается стабильность топ-10 (сколько из 10 лучших кандидатов остаются в топе). Результат визуализируется тепловой картой.

### Анализ распределения баллов

Гистограмма с KDE-оверлеем, отметками среднего и медианы, а также стандартным отклонением. Позволяет оценить нормальность распределения и выявить выбросы.

## 9. Справедливость и прозрачность

### Анализ справедливости

- **По городам**: средний балл и стандартное отклонение по каждому городу с визуализацией
- **По уровню образования**: сравнение баллов школьников, бакалавров и магистров
- **По опыту работы**: группы 0 лет, 1-2 года, 3+ лет
- **Коэффициент диспаратного воздействия**: отношение среднего балла группы к общему среднему. Флаг при значении <0.80 (правило 4/5 из трудового законодательства США)
- **Справедливость ИИ-детекции**: анализ того, не получают ли кандидаты из определённых городов непропорционально частые флаги ИИ-генерации

### Объяснимость (Explainability)

Для каждого кандидата система предоставляет:
- Детализацию баллов по 5 измерениям и суб-компонентам
- Текстовое обоснование с указанием методологии и фактических оснований
- Теги доказательной базы (evidence tags)
- NLP-метрики для каждого эссе
- Радарную диаграмму компетенций

### Human-in-the-loop

- Комиссия может скорректировать автоматическую оценку любого кандидата
- Корректировка требует текстового обоснования
- Возможность изменить статус кандидата вручную
- Корректированные баллы учитываются в рейтинге наравне с автоматическими

## 10. Ограничения системы

1. **Эвристическая детекция ИИ**: использует статистические маркеры (однородность предложений, переходные слова, биграммы), не гарантирует 100% точности. Возможны как ложноположительные, так и ложноотрицательные результаты.
2. **Ограниченные списки ключевых слов**: NLP-анализ мотивации, лидерства и роста основан на фиксированных списках русскоязычных ключевых слов, которые не покрывают все возможные формулировки.
3. **Шкала GPA**: нормализация в диапазоне 2.0-4.0. Кандидаты с GPA по другим шкалам (5-балльная, 100-балльная) требуют предварительной конвертации.
4. **Языковая поддержка**: система оптимизирована для русскоязычных текстов. Анализ эссе на других языках может быть неточным.
5. **Текстовое совпадение навыков**: оценка навыков основана на простом совпадении строк с эталонным списком. Контекст применения, уровень владения и актуальность не учитываются.
6. **Экспертные веса**: веса критериев настраиваются экспертом вручную, а не обучаются на исторических данных. Оптимальные веса могут отличаться для разных программ.
7. **Отсутствие невербальных данных**: модель не анализирует интервью, видеозаписи или другие невербальные сигналы.
8. **Количественный учёт рекомендаций**: учитывается только количество рекомендательных писем, но не их содержание и качество.
9. **Малые выборки**: при количестве кандидатов менее 20 статистические метрики (корреляции, KDE, справедливость) могут быть ненадёжными.
10. **Синтетические данные**: встроенный генератор создаёт реалистичные, но не репрезентативные данные. Результаты на демо-данных не экстраполируются на реальные заявки.
11. **Отсутствие временной динамики**: система оценивает кандидатов на момент подачи заявки и не учитывает прогресс или изменения в профиле.
12. **Зависимость от полноты данных**: качество оценки прямо зависит от полноты заполнения профиля. Отсутствующие данные приводят к нулевым суб-баллам.

---

## 11. Приватность и безопасность данных

- Все вычисления выполняются локально на устройстве пользователя
- Внешние API не вызываются. Модель работает полностью автономно
- Данные сессии хранятся только в оперативной памяти и не сохраняются на диск без явного запроса
- Экспорт результатов (JSON, CSV) выполняется только по инициативе пользователя
- Персональные данные кандидатов не передаются третьим сторонам
- Встроенный режим анонимизации заменяет имена на "Кандидат #N"
- Система не собирает метрики использования и не отправляет телеметрию

## 13. Сценарии использования

### Типовой рабочий процесс приёмной комиссии

1. **Загрузка данных**: импорт заявок кандидатов из CSV или JSON, либо генерация демо-набора для ознакомления
2. **Настройка модели**: корректировка весов критериев в соответствии с приоритетами программы. Проверка суммы весов (должна быть 1.0)
3. **Запуск оценки**: автоматическая оценка всех кандидатов с NLP-анализом эссе. Прогресс-бар отображает ход оценки
4. **Просмотр рейтинга**: таблица с фильтрами по имени, баллу, статусу. Экспорт в JSON/CSV
5. **Анализ профилей**: детальный просмотр каждого кандидата -- радар компетенций, суб-компоненты, NLP-метрики, обоснования
6. **Сравнение**: наложение радарных диаграмм 2-4 кандидатов, побудительное сравнение по измерениям
7. **Формирование шорт-листа**: автоматический отбор по порогу с возможностью исключения кандидатов с подозрительными эссе
8. **Корректировка**: ручная корректировка баллов с обоснованием (human-in-the-loop)
9. **Аналитика**: анализ распределений, корреляций, сравнение с базовыми моделями, валидация, проверка справедливости
10. **Экспорт**: выгрузка шорт-листа и полных результатов в JSON или CSV

## 14. API и интеграция

Система разработана как модульное приложение. Для интеграции с внешними системами:

### Программный интерфейс (Python)

```python
from models import CandidateProfile, ScoringConfig
from scoring import ScoringEngine
from utils import dict_to_candidate, export_results_to_json

# Загрузка кандидатов
candidates = [dict_to_candidate(data) for data in candidate_dicts]

# Настройка конфигурации
config = ScoringConfig(
    motivation_weight=0.25,
    leadership_weight=0.20,
    growth_weight=0.20,
    skills_weight=0.20,
    experience_weight=0.15,
)

# Оценка
engine = ScoringEngine(config)
for candidate in candidates:
    engine.score_candidate(candidate)
engine.rank_candidates(candidates)

# Формирование шорт-листа
shortlist = engine.generate_shortlist(candidates, threshold=70.0, max_count=20)

# Экспорт
json_output = export_results_to_json(shortlist)
```
